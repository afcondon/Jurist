A double pendulum, as the DAE it actually is

In Cartesian coordinates the rod tensions are algebraic variables — no time derivative, closed by rigidity constraints. That makes this a differential-algebraic system, which scipy.solve_ivp and every JS ODE library structurally cannot solve. ModelingToolkit derives the index-1 form symbolically; an implicit stiff solver holds the rods rigid to ~10⁻⁸ through fully chaotic motion.

The PureScript you write

-- differential state · algebraic vars · parameters,
-- all row-typed
type DPState =
  ( x1 :: NumExpr, y1 :: NumExpr, x2 :: NumExpr, y2 :: NumExpr
  , u1 :: NumExpr, v1 :: NumExpr, u2 :: NumExpr, v2 :: NumExpr )
type DPAlg = ( t1 :: NumExpr, t2 :: NumExpr )  -- rod tensions

doublePendulum :: DAESpec DPState DPAlg DPParams
doublePendulum =
  daeSystem
    -- the differential part
    ( \s a p ->
        let ac = accels s a p
        in
          { x1: s.u1, y1: s.v1, x2: s.u2, y2: s.v2
          , u1: ac.ax1, v1: ac.ay1, u2: ac.ax2, v2: ac.ay2
          }
    )
    -- one rigidity constraint per tension
    -- (|p|² = L², differentiated twice)
    ( \s a p ->
        let ac = accels s a p
        in
          { t1: s.u1 * s.u1 + s.v1 * s.v1
              + s.x1 * ac.ax1 + s.y1 * ac.ay1
          , t2: (s.u2 - s.u1) * (s.u2 - s.u1)
              + (s.v2 - s.v1) * (s.v2 - s.v1)
              + (s.x2 - s.x1) * (ac.ax2 - ac.ax1)
              + (s.y2 - s.y1) * (ac.ay2 - ac.ay1)
          }
    )

accels gives the Cartesian accelerations in terms of the tensions, written once and shared by both lambdas. MTK solves the 2×2 tension system symbolically — never written in PureScript.
