Structural patterns, found by homomorphism search

A module dependency graph and three structural motifs, described once in PureScript. Catlab finds every occurrence as the set of graph homomorphisms pattern → graph — subgraph matching generalized to any schema, the category doing the searching. The same engine finds circular dependencies, mutual dependencies, and diamonds — the machinery a Minard could use to flag architectural motifs in a code graph.

-- A → B means "module A depends on module B"
depGraph :: Graph
depGraph =
  { nodes: [ "Core", "Parser", "AST", "Eval"
           , "Pretty", "Codegen", "Cli", "Utils" ]
  , edges:
      [ edge 6 0, edge 6 1   -- Cli → Core, Parser
      , edge 1 2, edge 2 3   -- Parser → AST → Eval

      , edge 4 5, edge 5 4   -- Pretty ⇄ Codegen
      , edge 0 7, edge 1 7   -- Core, Parser → Utils
      , edge 5 2              -- Codegen → AST
      ]
  }

A match is a vertex map; PureScript canonicalises the rotations/orderings the search returns down to distinct occurrences.
