Architectural motifs over a typed schema

A typed code graph — modules, functions typed into modules, calls — as an instance of a Catlab ACSet schema. Each motif is a small instance of the same schema, and its type structure encodes the rule: two distinct pattern-modules match only cross-module structure. The typed-graph analogue of Hylograph's graphs, and the machinery a Minard could use to express architectural rules.

The PureScript you write

-- schema: Func —in_mod→ Module · Call —src,tgt→ Func
codeGraph :: CodeGraph
codeGraph =
  { modules: [ "Auth", "Db", "Api", "Util" ]
  , funcs:
      [ fn "login" 0, fn "hashPwd" 0    -- Auth
      , fn "query" 1, fn "connect" 1    -- Db
      , fn "handleReq" 2, fn "route" 2  -- Api
      , fn "log" 3, fn "validate" 3     -- Util
      ]
  , calls:
      [ edge 0 2, edge 0 1, edge 0 6
      , edge 2 3, edge 2 6
      , edge 4 0, edge 4 5, edge 5 2
      , edge 7 6, edge 4 7
      ]
  }

-- two distinct pattern-modules ⇒ only
-- cross-module structure can match
motifs :: Array Motif
motifs =
  [ { name: "Cross-module call"
    , shape: "f @A → g @B", nMod: 2
    , fnMod: [ 0, 1 ], calls: [ edge 0 1 ] }
  , { name: "Shared dependency"
    , shape: "f @A, g @B → h @C", nMod: 3
    , fnMod: [ 0, 1, 2 ]
    , calls: [ edge 0 2, edge 1 2 ] }
  ]

ms <- matches codeGraph mo   -- typed search
