Provably optimal

A 0/1 knapsack described once in PureScript. The obvious greedy heuristic (computed in pure PureScript) grabs the best value-per-weight item and stalls. Julia's JuMP/HiGHS mixed-integer solver doesn't guess — branch-and-bound proves the global optimum, closing the gap to zero. There is no MILP solver in the JS/WASM world to hand a model to.

The PureScript you write

let
  items =
    [ { name: "Telescope", w: 6.0, v: 12.0 }
    , { name: "Camera",    w: 5.0, v: 9.0 }
    , { name: "Tripod",    w: 5.0, v: 9.0 }
    , { name: "Drone",     w: 7.0, v: 8.0 }
    ]
  weights = map _.w items
  values  = map _.v items
  cap     = 10.0

  -- the myopic baseline, in pure PureScript
  greedy  = greedyValue weights values cap

-- the proof, on the Julia runtime
opt <- knapsack weights values cap

Greedy is the developer's mental model; knapsack builds a JuMP model with binary variables and lets HiGHS close the optimality gap.
