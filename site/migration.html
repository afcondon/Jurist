Functorial data migration

A function-level call graph, migrated functorially to its module dependency graph — a Δ migration along the in_mod functor, where every call f → g becomes a module edge in_mod(f) → in_mod(g). Not a hand-written fold; a migration functor with categorical guarantees. The result is a plain graph — exactly the shape Hylograph lays out and renders.

The PureScript you write

codeGraph :: CodeGraph
codeGraph =
  { modules: [ "Auth", "Db", "Api", "Util" ]
  , funcs:
      [ fn "login" 0, fn "hashPwd" 0
      , fn "query" 1, fn "connect" 1
      , fn "handleReq" 2, fn "route" 2
      , fn "log" 3, fn "validate" 3
      ]
  , calls:
      [ edge 0 2, edge 0 3   -- Auth → Db   (×2)
      , edge 0 6, edge 1 6   -- Auth → Util (×2)
      , edge 2 6              -- Db → Util   (×1)
      , edge 4 0, edge 5 1   -- Api → Auth  (×2)
      , edge 5 2, edge 4 2
      , edge 4 3              -- Api → Db    (×3)
      , edge 4 6              -- Api → Util  (×1)
      , edge 0 1, edge 2 3   -- intra-module
      ]
  }

-- the migration: V ↦ Module, E ↦ Call,
-- src ↦ src⨟in_mod, tgt ↦ tgt⨟in_mod
raw <- moduleGraph codeGraph

-- PureScript aggregates the migrated multigraph
-- into weighted module dependencies
agg = aggregate raw

Catlab returns one module edge per call; the weights (×2, ×3) are the multiplicities — coupling strength for free.
