Portability is the development story; the Julia-only interpreters are the reason to deploy. And where two runtimes run the same interpreter, they agree to the last float — the Lorenz receipts at the top of the page.

The unglamorous part

Underneath is purejl, a from-scratch PureScript→Julia backend. It consumes CoreFn — the compiler's fully-elaborated core — so the whole language arrives by construction: typeclasses, row types, ADTs, the works (the row-typed SystemSpec machinery above is RowList-driven typeclass derivation, running on Julia). Core-library coverage: 50 built-in FFI shims (prelude, arrays incl. ST, strings, records, integers, numbers, …; Data.String.Regex is a deliberate loud-failure stub). Verified by differential testing against the JS backend as reference semantics: 422 of 426 outputs byte-identical, the other 4 documented representation choices (UTF-16 code units vs. codepoints).
