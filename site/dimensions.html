Dimensions in the type system

Dimensional analysis, twice over. A Quantity carries its physical dimension in the type — exponents of mass, length, time as type-level integers. Multiplication adds exponents, division subtracts them, √ halves them, all solved by the compiler — so adding a spring constant to a mass does not compile, on any backend, browser included. And at system scale, unit annotations ride the same typed rows: ModelingToolkit · DynamicQuantities prove every equation dimensionally consistent — or refuse, naming the term and the clashing units.

The PureScript you write

springK :: Quantity 1 0 (-2)   -- kg·s⁻², i.e. N/m
springK = newtonsPerMeter 5.2

bobMass :: Quantity 1 0 0
bobMass = kilograms 0.35

-- |/| subtracts exponents, √ halves them: this only
-- compiles because √(kg / kg·s⁻²) works out to s¹
period :: Quantity 0 0 1
period = scalar (2.0 * pi)
           |*| qSqrt (bobMass |/| springK)
does not compile nonsense = springK |+| bobMass [ERROR] TypesDoNotUnify Could not match type 0 with type -2 while trying to match type Quantity 1 0 0 with type Quantity 1 0 -2

The exponents are Prim.Int type-level integers; they reflect to plain values (Data.Reflectable) and cross the seam once — the same dimension record DynamicQuantities consumes.

What Julia computes — and refuses

Two layers of the same guarantee: the types reject mis-dimensioned expressions before the program exists; the runtime refuses the mistakes only it can see — like a conversion target that arrives as a string.

Units on a whole system

-- a falling body with linear drag; c has units 1/s
fallingBody = system \s p ->
  { y: s.v, v: negate p.g - p.c * s.v }

-- annotations are records matching the system's rows:
-- a unit for a misspelt variable is a compile error
validateUnits fallingBody
  { y: "m", v: "m/s" }
  { g: "m/s^2", c: "1/s" }

-- the same body, drag mistyped as c·v² — the SAME c.
-- NumExpr is dimensionless, so this still type-checks;
-- only the production interpreter can catch it:
fallingBodyBad = system \s p ->
  { y: s.v, v: negate p.g - p.c * s.v * s.v }

On Node and the BEAM this verb answers Deferred; on Julia, MTK attaches the units as variable metadata and validates at system construction.
